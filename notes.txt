============================================================
PROJET GRAPHES ET OPEN DATA — RÉSEAU D’ACTEURS TMDB
Notes de suivi du projet
============================================================

Sujet :
Analyse temporelle et communautaire d’un réseau d’acteurs à partir du dataset ouvert TMDB 5000 Movie Dataset.

Dataset :
- tmdb_5000_movies.csv
- tmdb_5000_credits.csv

Objectif général :
Créer un graphe d’acteurs à partir de données open data, analyser les collaborations entre acteurs, détecter des communautés, exploiter les années, les genres, les films et proposer des visualisations ainsi qu’un système simple de recommandation.

Principe du graphe :
- Nœud = acteur
- Arête = deux acteurs ont joué ensemble dans au moins un film
- Poids de l’arête = nombre de films joués en commun

Fichier : src/graph_build.py
Commande : py -3.12 -m src.graph_build

Phrase d’explication :
Nous avons reconstruit un graphe par période afin d’analyser l’évolution des collaborations entre acteurs dans le temps.

6. VÉRIFICATION FILMS / COMMUNAUTÉS
Fichier : src/film_community_check.py
Commande : py -3.12 -m src.film_community_check
============================================================

Objectif du professeur :
Lister les films afin de vérifier si les acteurs appartiennent aux mêmes films ou aux mêmes communautés.

Objectif du script :
Analyser chaque film pour savoir :
- quels acteurs sont dedans
- à quelles communautés appartiennent ces acteurs
- quelle communauté domine le film
- si le film est cohérent avec une communauté
- ou s’il sert de pont entre plusieurs communautés

Fichier généré :
outputs/reports/movie_community_check.csv

Résultat obtenu lors de l’exécution :
- Nombre de films analysés : 4089
- Films cohérents avec une communauté : 2552
- Films ponts / mixtes : 1537

Interprétation :
Sur 4089 films analysés, 2552 films ont une majorité d’acteurs appartenant à une même communauté. Cela montre que les communautés détectées par Louvain correspondent souvent à de vrais groupes de collaboration.
Les 1537 autres films sont plus mixtes et peuvent être vus comme des films-ponts entre plusieurs communautés.

Phrase d’explication :
Pour aller au-delà de la simple détection de communautés, nous avons relié chaque film aux communautés de ses acteurs. Cela permet de vérifier si les films sont principalement associés à une seule communauté ou s’ils mélangent plusieurs groupes d’acteurs.

Phrase complémentaire :
Les films dont les acteurs appartiennent à plusieurs communautés peuvent être interprétés comme des films-ponts entre différents sous-réseaux.

8. MENU INTERACTIF
Fichier : src/cli_menu.py
Commande : py -3.12 -m src.cli_menu
============================================================

Objectif :
Centraliser toutes les fonctionnalités du projet dans un seul menu utilisable en ligne de commande.

Phrase d’explication :
Le menu permet d’utiliser le projet comme une petite application de démonstration. L’utilisateur peut relancer les analyses, générer les visualisations et tester les recommandations sans exécuter manuellement chaque fichier Python.

10. CARTE MONDE INTERACTIVE
Fichier : src/viz_world_map.py
Commande : py -3.12 -m src.viz_world_map
============================================================

Librairie utilisée :
Folium

Installation :
py -3.12 -m pip install folium

Objectif :
Créer une carte du monde interactive afin de visualiser les acteurs selon une dimension géographique.

Fichiers générés dans outputs/web :
- world_map_1990_1999.html
- world_map_2000_2009.html
- world_map_2010_2019.html

Fonctionnalités :
- Chaque acteur est positionné sur une carte du monde
- La position est basée sur le pays dominant de production des films dans lesquels il apparaît
- Au clic ou au survol, on peut voir :
  - nom de l’acteur
  - communauté
  - degré total
  - pays dominant
  - genres dominants
  - films de la période
- Les liens affichent :
  - le nombre de films en commun
  - les noms des films en commun

Remarque importante :
Le dataset TMDB ne fournit pas directement le lieu de naissance des acteurs. Les acteurs sont donc positionnés selon le pays de production le plus fréquent des films où ils apparaissent.
C’est une approximation logique et défendable.

Phrase d’explication :
Le degré total correspond au nombre d’acteurs différents avec lesquels un acteur a collaboré dans toute la période. Pour rendre la carte lisible, nous affichons un sous-graphe limité aux acteurs les plus centraux.

Choix technique :
Pour éviter la confusion liée aux liens cachés, nous pouvons afficher tous les liens du sous-graphe et contrôler la lisibilité avec le paramètre top_n.

13. PHRASES IMPORTANTES POUR LE RAPPORT / ORAL
============================================================

Phrase 1 :
Nous ne nous sommes pas limités à construire un graphe global. Nous avons exploité les fichiers CSV pour introduire le temps, les genres, les films, les pays de production, les communautés et une logique de recommandation.

Phrase 2 :
Les communautés détectées par Louvain n’ont pas de noms officiels dans le dataset. Elles sont identifiées par un numéro technique, puis interprétées à partir des genres dominants et des films associés.

Phrase 3 :
Les films-ponts sont des films dont les acteurs appartiennent à plusieurs communautés. Ils permettent de relier différents sous-réseaux d’acteurs.

Phrase 4 :
Le fichier .gpickle est un format interne destiné à Python. Pour rendre le graphe directement visible par l’utilisateur, nous générons aussi des fichiers CSV, GraphML et HTML interactifs.

Phrase 5 :
La carte monde ne représente pas le lieu de naissance des acteurs. Elle positionne les acteurs selon le pays dominant de production des films dans lesquels ils apparaissent, car cette information est disponible dans le dataset.

Phrase 6 :
Le menu interactif permet de transformer le projet en application de démonstration, utilisable sans exécuter chaque script séparément.
